# 福彩AI助手 — 开发需求清单
> 基于《福彩AI助手产品方案（V3.0最终整合版）》提取  
> 版本：V1.0  
> 日期：2026-06-24

---

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 产品名称 | 福彩AI助手 |
| 产品定位 | 纯工具型福彩信息查询小程序（微信小程序 + H5备用入口） |
| 核心原则 | **三不原则**：不选号、不代销、不收集个人信息 |
| 目标用户 | 福彩彩民（C端）、福彩站点（B端推广方） |
| 商业模式 | 免费基础功能 + Pro高级订阅（¥9.9/月） + 广告变现 + 远期增值服务 |
| 开发周期 | MVP：2个月；首年路线图：V1.0 → V1.5 → V2.0 → V2.5 |

### 4.4 数据来源

## 五、合规与安全需求清单

### 5.1 绝对禁止（9项红线）

| 序号 | 禁止项 | 法规依据 |

| 4 | 收集个人信息（手机号、身份证、地理位置等） | 《个人信息保护法》第五条、第六条 |
| 5 | 生成注单/投注方案 | 《彩票管理条例实施细则》第七条 |

| 9 | 线上选号→线下出票闭环 | 《彩票管理条例实施细则》第七条 |

### 5.2 AI合规过滤（三层）

| 层级 | 功能 | 说明 |
|------|------|------|
| 第一层 | 意图识别拦截 | 违规请求直接返回标准拒绝话术，不进入后续流程 |
| 第二层 | 生成结果审查 | 关键词匹配 + 语义分析，拦截预测/诱导/虚构内容 |
| 第三层 | 人工审核兜底 | 日志记录 + 5%抽样人工审核 + 用户举报机制 + 24小时响应 |
